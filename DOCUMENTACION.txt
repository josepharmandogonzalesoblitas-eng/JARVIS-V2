 1. RESUMEN DEL PROYECTO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

JARVIS V2 es un asistente virtual conversacional impulsado por Google Gemini
(modelo multimodal) que opera principalmente a través de Telegram.

Su misión va más allá de responder preguntas: es un "segundo cerebro" y
compañero estratégico que conoce al usuario profundamente, anticipa sus
necesidades, detecta sus emociones y actúa proactivamente.

PILARES DEL SISTEMA:
  • Cerebro IA         → Google Gemini (texto + voz + visión)
  • Memoria Híbrida    → JSON estructurado + ChromaDB vectorial
  • Motor Emocional    → Detección de crisis, celebración de logros
  • Modos de Diálogo   → Escucha profunda, foco, silencio, terapeuta
  • Voz (TTS)          → gTTS (sin costo, sin API key)
  • Gráficos           → Matplotlib (energía, proyectos, resumen mensual)
  • Clima              → OpenWeatherMap (opcional)
  • Visión             → Gemini Vision API (análisis de fotos)
  • CRON Inteligente   → 10+ tareas automáticas con triggers contextuales

 3. SISTEMA DE MEMORIA HÍBRIDA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

JARVIS usa tres capas de memoria con diferentes características:

┌─────────────────────────────────────────────────────────────────┐
│ CAPA 1: MEMORIA EN SESIÓN (RAM)                                 │
│  • Historial reciente de la conversación actual (últimos 12)    │
│  • Estado de modos de conversación activos                      │
│  • Acceso: O(1) — inmediato                                     │
│  • Durabilidad: Se pierde al reiniciar (intencional)            │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ CAPA 2: MEMORIA ESTRUCTURADA (JSON en disco)                    │
│  • persona.json     → Nombre, edad, profesión, preferencias     │
│  • entorno.json     → Ubicación, timezone, personas clave       │
│  • proyectos.json   → Proyectos activos con tareas y estado     │
│  • contexto.json    → Recordatorios pendientes, rutinas         │
│  • bitacora.json    → Registro diario de energía y estado ánimo │
│  • estado_emocional.json → [NUEVO] Crisis, logros, follow-ups   │
│  • Acceso: Asíncrono con escritura atómica (tmp + fsync)        │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ CAPA 3: MEMORIA SEMÁNTICA (ChromaDB vectorial)                  │
│  • Hechos de vida importantes (cumpleaños, alergias, logros)    │
│  • Embedding: Google Generative AI Embeddings                   │
│  • Búsqueda: Por similitud semántica (cosine similarity)        │
│  • Auto-migración si hay conflicto de embedding function        │
└─────────────────────────────────────────────────────────────────┘

INTENCIONES DE MEMORIA (Lo que la IA detecta automáticamente):
  actualizar_nombre          → "Me llamo Joseph"
  actualizar_edad            → "Tengo 30 años"
  actualizar_profesion       → "Soy ingeniero"
  actualizar_preferencia     → "Me gusta el azul / los gatos / etc."
  actualizar_rutina          → "Hago ejercicio a las 7am"
  actualizar_persona_clave   → "Mi esposa se llama María"
  actualizar_estado_animo    → "Hoy me siento cansado / energía 8"
  nuevo_recordatorio         → "Recuérdame comprar leche"
  nuevo_recuerdo_largo_plazo → "Mi hijo nació el 16 de febrero"
  guardar_conversacion_profunda → [NUEVO] Tema con seguimiento automático
  registrar_logro            → [NUEVO] Logro importante del usuario

 4. MÓDULOS NUEVOS (V3.0)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

─────────────────────────────────────────────────────────────────
 4.1 MOTOR EMOCIONAL (emotion_engine.py)
─────────────────────────────────────────────────────────────────
Analiza cada mensaje del usuario en tiempo real buscando patrones.

DETECCIÓN DE CRISIS (2 niveles):
  Nivel 1 → Palabras de tristeza/soledad ("solo", "triste", "mal")
             → JARVIS responde con empatía profunda
             → Si persiste 3+ días: envía alerta proactiva
  Nivel 2 → Palabras de desesperanza ("no puedo más", "no vale la pena")
             → Respuesta inmediata de seguridad + recursos de apoyo

CELEBRACIÓN DE LOGROS:
  → Detecta cuando el usuario completa algo importante
  → Genera mensaje de celebración personalizado
  → Registra el logro en estado_emocional.json
  → Guarda en memoria vectorial para referencia futura

FOLLOW-UP AUTOMÁTICO:
  → Cuando la IA detecta un tema importante (salud, relación, meta),
    guarda una "conversación profunda" con fecha de seguimiento
  → El CRON verifica diariamente si hay follow-ups pendientes
  → Ejemplo: "La última vez hablamos de tu preocupación por X.
              ¿Fuiste al médico como planeabas?"

TRIGGERS DE METAS OLVIDADAS:
  → Registra la fecha de última mención de cada meta
  → Si pasan 5+ días sin mencionar una meta activa:
    → Mensaje espontáneo: "Hace días no mencionas tu meta de [X].
                           ¿Perdiste el interés o solo falta tiempo?"

─────────────────────────────────────────────────────────────────
 4.2 MODOS DE CONVERSACIÓN (conversation_state.py)
─────────────────────────────────────────────────────────────────
Gestiona el estado activo de la conversación. Se activa por comandos
de Telegram o cuando la IA detecta la necesidad.

MODO NORMAL (default):
  → Comportamiento estándar

MODO ESCUCHA PROFUNDA (/escucha):
  → Alta empatía, mantiene el hilo emocional
  → No cambia de tema ni ofrece soluciones no pedidas
  → Dura 5 intercambios, luego vuelve a Normal automáticamente
  → La IA puede activarlo si detecta que el usuario necesita ser escuchado

MODO TRABAJO PROFUNDO (/foco [minutos]):
  → JARVIS no envía notificaciones proactivas (CRON silenciado)
  → Respuestas muy breves (1 frase)
  → Al terminar el tiempo: "¿Qué lograste en esta sesión?"

MODO SILENCIOSO (/silencio [minutos]):
  → JARVIS sigue respondiendo si se le escribe
  → Pero NO envía mensajes del CRON (check-ins, recordatorios)
  → Ideal para horas de trabajo o descanso

MODO TERAPEUTA (/terapeuta):
  → Sesión estructurada de 5 preguntas (basada en CBT + Journaling)
  → Preguntas: Victoria semanal, Desafío y aprendizaje, Bienestar físico,
               Tarea evitada, Calificación global de la semana
  → También se activa automáticamente cada domingo a las 20:00

─────────────────────────────────────────────────────────────────
 4.3 TEXT-TO-SPEECH — RESPUESTAS EN VOZ (tool_tts.py)
─────────────────────────────────────────────────────────────────
Convierte las respuestas de texto en notas de voz de Telegram.

  • Tecnología: gTTS (Google Text-to-Speech)
  • Costo: GRATUITO (sin API key necesaria)
  • Formato: MP3 → enviado como nota de voz en Telegram
  • Idioma: Español por defecto (configurable en .env)
  • Activación: Comando /tts (toggle on/off)

CONFIGURACIÓN EN .env (opcional):
  TTS_IDIOMA="es"       → es, en, pt, fr, de...
  TTS_LENTA="false"     → "true" para voz más lenta

─────────────────────────────────────────────────────────────────
 4.4 CLIMA EN TIEMPO REAL (tool_weather.py)
─────────────────────────────────────────────────────────────────
Obtiene el clima actual y pronóstico usando OpenWeatherMap.

  • API: OpenWeatherMap (plan gratuito: 1000 llamadas/día)
  • Requiere: OPENWEATHER_API_KEY en el .env
  • Ciudad por defecto: OPENWEATHER_CIUDAD en el .env (ej: "Lima")
  • Si no está configurado: el sistema funciona normal sin esta función

USO DESDE TELEGRAM:
  → Comando /clima [ciudad]     → Clima de cualquier ciudad
  → "¿Cómo está el clima hoy?" → JARVIS detecta y usa la herramienta

INTEGRACIÓN CON CRON:
  → El check-in matutino (08:00) incluye el clima si está configurado
  → Sugerencias contextuales: "Hoy llueve. ¿Quieres reprogramar tu
    rutina de ejercicio al gimnasio?"

─────────────────────────────────────────────────────────────────
 4.5 GRÁFICOS VISUALES (tool_graphs.py)
─────────────────────────────────────────────────────────────────
Genera imágenes con datos del usuario y las envía por Telegram.

  • Tecnología: Matplotlib (sin pantalla, backend Agg)
  • Formato de salida: PNG (enviado como foto en Telegram)

TIPOS DE GRÁFICOS:
  1. Gráfico de Energía (/progreso)
     → Línea de nivel de energía de los últimos N días
     → Colores: verde (7-10), amarillo (4-6), rojo (1-3)

  2. Progreso de Proyecto
     → Barras de tareas completadas vs. pendientes por proyecto
     → Disponible cuando la IA detecta que pides el progreso de un proyecto

  3. Resumen Mensual (/mes)
     → Scatter plot con energía y estado de ánimo del mes actual

USO DESDE TELEGRAM:
  → Comando /progreso         → Gráfico de energía (7 días)
  → Comando /mes              → Resumen visual del mes
  → "Muéstrame mi progreso"   → La IA genera el gráfico correspondiente

─────────────────────────────────────────────────────────────────
 4.6 VISIÓN ARTIFICIAL — ANÁLISIS DE IMÁGENES (cerebro.py)
─────────────────────────────────────────────────────────────────
JARVIS puede analizar fotos enviadas por Telegram usando Gemini Vision.

CASOS DE USO:
  • Foto de pizarra/notas     → Extrae tareas y las agenda
  • Foto de documento/factura → Extrae información relevante
  • Foto en el gym/corriendo  → Detecta el logro y lo celebra
  • Foto con texto            → Transcribe y actúa en consecuencia
  • Imagen enviada como archivo → También procesada (MIME detection)

CÓMO USARLO:
  → Simplemente envía una foto por Telegram (con o sin caption)
  → JARVIS la descarga, la envía inline a Gemini Vision y responde
  → No requiere comandos especiales

─────────────────────────────────────────────────────────────────
 4.7 CRON INTELIGENTE V2 (cron.py)
─────────────────────────────────────────────────────────────────
El scheduler autónomo ahora tiene 10+ tareas con lógica contextual.

TAREAS DIARIAS:
  08:00 → Check-in matutino (saludo + rutinas + clima + recordatorios)
  09:00 → Verificar follow-ups de conversaciones profundas pendientes
  11:00 → Detectar si hay patrón emocional negativo acumulado (3+ días)
  14:00 → Check-in de mediodía (progreso del día + recordatorios tarde)
  20:30 → Check-in nocturno (cierre del día + reflexión de bienestar)
  22:00 → Resumen diario (pendientes, energía, motivación)
  03:00 → Backup automático de toda la memoria

TAREAS SEMANALES:
  Lun 08:30 → Análisis de patrones de energía de la semana pasada
  Mar 10:00 → Trigger de metas: ¿hay metas sin mencionar por 5+ días?
  Mié 10:00 → Sugerencia no solicitada (búsqueda web basada en intereses)
  Jue 10:00 → Trigger de metas (segunda verificación semanal)
  Dom 10:00 → Seguimiento de metas a largo plazo
  Dom 18:00 → Validación de rutinas ("¿sigues yendo al gym?")
  Dom 20:00 → Sesión terapeuta guiada (5 preguntas de reflexión)

RESPETO DE MODOS:
  → Si el modo es SILENCIOSO o TRABAJO PROFUNDO, el CRON no envía
    mensajes proactivos hasta que el modo se desactive

ALARMAS DINÁMICAS:
  → "Avísame a las 15:30 que tengo reunión" → CRON agenda esa alarma
  → Se resetean automáticamente cada día a las 03:00 (con el backup)

  [ ] PRUEBA 1 — Seguridad Zero-Trust

 11. TROUBLESHOOTING (ERRORES FRECUENTES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ERROR: PermissionError: '/app/LOGS/jarvis_v2.log'
  → El volumen de LOGS tiene permisos de root
  → Solución:
    ssh root@IP "chmod -R 777 /app/jarvis_v2/LOGS /app/jarvis_v2/MEMORIA"
    docker restart jarvis_v2_bot

ERROR: ChromaDB embedding function conflict
  → La colección vectorial fue creada con un embedding diferente al actual
  → Solución: Automática desde V3.0 (la colección se re-crea sola)
  → Si persiste: ssh root@IP "rm -rf /app/jarvis_v2/MEMORIA/vector_db"
    Y reiniciar el contenedor (se re-inicializa vacía)

ERROR 403: Google Calendar/Tasks API not enabled
  → Solución:
    1. Ir a https://console.cloud.google.com/
    2. APIs & Services > Library
    3. Buscar "Google Calendar API" → Enable
    4. Buscar "Google Tasks API" → Enable
    5. Esperar 5 minutos y reintentar

ERROR: Gemini API Key inválida / timeout
  → El sistema activa Graceful Degradation automáticamente
  → Responderá con herramientas locales (hora, estado del sistema)
  → Verificar la API key: ssh root@IP "nano /app/jarvis_v2/.env"

ERROR: El bot no responde a nadie
  → Verificar que el contenedor está corriendo:
    docker ps --filter name=jarvis_v2_bot
  → Ver logs: docker logs jarvis_v2_bot --tail 30
  → Reiniciar: docker restart jarvis_v2_bot

ERROR: "Application started" pero no hay respuesta en Telegram
  → Verificar TELEGRAM_USER_ID en el .env
  → El ID debe ser el tuyo (no el ID del bot)
  → Obtener tu ID: escribe a @userinfobot en Telegram

FILOSOFÍA DE PRUEBAS (¿Por qué los tests usan Mocks?):
  Los 47 tests automatizados usan "Mocks" para las APIs externas.
  Esto es correcto y por diseño:
  • Tests unitarios/integración → Prueban NUESTRO código, no el de Google
    Son rápidos, no gastan cuota de API, no dependen de internet
  • Tests E2E reales → Prueban el sistema completo en producción
    Encuentran errores del mundo real (credenciales, cambios de API)
  Ambas pruebas son complementarias, no se reemplazan.
